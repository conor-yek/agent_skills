# SDK 文档编写规范

本文件是 AI 生成/修复 SDK 文档的唯一格式规范来源。所有文档类型共享通用规则，按类型套用对应模板。

---

## 通用规则（所有类型共享）

### 语言

- 正文描述使用中文
- API 名称、配置项、变量名、日志输出使用英文
- 代码注释使用中文

### 代码块

- 必须标注语言：`c`、`kconfig`、`bash`、`ini`、`text`、`json`、`yaml`、`cmake`、`python`、`rst`
- 不可使用自造语言名或空标注紧跟内容（会产生 Sphinx warning）

### 术语

- 设备名大写：ADC0、GPIOA、UART0
- API 函数用反引号加括号：`lisa_gpio_configure()`
- CONFIG_ 宏用反引号：`CONFIG_LISA_GPIO`
- 枚举值用反引号：`LISA_GPIO_INPUT`
- 模块名正式写法：LISA GPIO（不是 lisa_gpio / GPIO / lisa gpio）
- 技术术语大写：DMA 模式、CPU 占用（不是 dma 模式、cpu 占用）

### 列表

- 有序列表：步骤、注意事项
- 无序列表：特性、引脚、配置项

### 强调

- **粗体**：关键词、重要参数值、引脚编号
- 反引号：代码元素（函数、变量、配置项）

### 标题

- 不使用 emoji 标题（基于实际文档风格分析，已发布文档均不使用 emoji）
- H1 只有一个（文档标题），H2 为主要章节，H3 为子章节，避免 H4+

### 文档格式

- 优先使用 Markdown (.md)
- RST (.rst) 仅在需要复杂文档结构时使用
- 两种格式章节结构和内容要求完全相同

## 示例文档模板（samples/ + demos/）

适用路径：`samples/<category>/<name>/README.md`、`demos/<name>/README.md`

### 必选章节及顺序

| 序号 | 章节 | 标题格式 |
|------|------|----------|
| 1 | 标题 | `# {模块名} {功能}示例` |
| 2 | 功能说明 | `## 功能说明` |
| 3 | 硬件连接 | `## 硬件连接` |
| 4 | 示例内容/步骤 | `## 示例内容` 或 `## 示例步骤` |
| 5 | 编译 | `## 编译` |
| 6 | 烧录 | `## 烧录` |
| 7 | 预期输出 | `## 预期输出` |

### 推荐章节

- `## 核心 API`（Devices/Modules 类型强烈推荐）
- `## 关键代码`
- `## 注意事项`（3-5 项，有序列表，关键词粗体）

### 可选章节

- `### 新特性`（高级示例，作为功能说明的 H3 子章节）
- `## 配置说明`
- `## 使用场景`
- `## 验证方法`

### 章节细则

**标题 (H1)**：`# {模块名称} {功能描述}示例{（可选模式说明）}`
- 模块名使用正式名称（LISA GPIO、系统堆管理、HTTP 客户端）
- 示例：`# LISA GPIO 基础输出示例`、`# LISA UART 同步接收示例（DMA 模式）`

**功能说明**：1-2 句话，40-200 字，说明"做什么"和"怎么做"

**硬件连接**：
- 有外部连接：无序列表，引脚编号粗体 `- **PB09**: LED 控制引脚`
- 无外部连接：`无需外部连接，{模块}为芯片内部资源。`

**示例内容/步骤**：有序列表，3-6 个步骤

**编译 + 烧录**：独立的两个 H2 章节，推荐使用 RST 引用格式：
```markdown
## 编译

​```{eval-rst}
.. include:: /sample_build.rst
​```

## 烧录

​```{eval-rst}
.. include:: /sample_flash.rst
​```
```

**预期输出**：代码块展示真实输出，多源用粗体标题分隔

**核心 API**：表格格式 `| API | 说明 |`，3-8 个，按调用顺序排列

**关键代码**：C 代码块，关键处加中文注释

**注意事项**：有序列表，每项 `**关键词**: 说明`

### 示例类型差异

| 类型 | 路径特征 | 额外要求 |
|------|----------|----------|
| 设备驱动 | `samples/drivers/devices/` | 硬件连接必须详细，核心 API 必选 |
| 组件模块 | `samples/modules/` | 核心 API 必选，关键代码必选 |
| 网络示例 | `samples/network/` | 可增加测试环境准备章节 |
| HAL 驱动 | `samples/drivers/hal/` | 通常不含核心 API 章节 |

## 模块文档模板（drivers/ + components/ + modules/）

适用路径：`drivers/<lisa_xxx>/README.md`、`components/<name>/README.md`

### 必选章节及顺序

| 序号 | 章节 | 标题格式 |
|------|------|----------|
| 1 | 标题 | `# {名称} 驱动` 或 `# {名称} 组件` |
| 2 | 功能特性 | `## 功能特性` |
| 3 | 配置选项 | `## 配置选项` |
| 4 | API 接口 | `## API 接口` |
| 5 | 使用示例 | `## 使用示例` 或 `## 使用方法` |
| 6 | 注意事项 | `## 注意事项` |

### 推荐章节

- `## 硬件配置`（驱动类，含引脚复用配置）
- `## 文件说明`
- 示例程序链接（指向 `samples/` 下对应示例）

### 可选章节

- 详细参数说明（如配置标志位、事件类型）
- 配置宏说明
- 返回值说明

### 章节细则

**标题 + 简介**：
```markdown
# ADC 驱动

基于 lisa_device 框架的 ADC 设备驱动，为 ARCS 平台提供统一的模拟信号采样接口。
```

**功能特性**：无序列表，`**粗体**` 突出特性名称，5-8 项，按重要性排序

**配置选项**：说明配置文件位置（prj.conf），使用 `kconfig` 代码块

**API 接口**：按功能分类（H3），展示函数原型，使用 `c` 代码块

**使用示例**：完整可运行代码，包含头文件和错误处理，从简单到复杂，至少 2-3 个场景

**硬件配置**（驱动类）：
- 必含引脚复用配置（pinmux）
- 说明定义位置、声明位置、调用时机

**注意事项**：有序列表，`**粗体**` 突出要点，包含使用限制、常见错误、线程安全说明

| 5 | 命令参数 | `## 命令参数` |

### 推荐章节

- `## 使用示例`
- `## 常见问题`

### 可选章节

- `## 配置选项`
- `## 输出说明`
